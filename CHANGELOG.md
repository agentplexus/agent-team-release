## [0.3.0] - 2026-01-12

### Highlights

- Complete transformation from pre-push validation tool to release automation platform with workflow engine, Claude Code integration, and multi-platform plugin support

### Added

- CLI restructure with Cobra framework and 7 commands: `check`, `validate`, `release`, `changelog`, `readme`, `roadmap`, `version`
- Release workflow engine with 9-step process: validate → check → changelog → commit → push → CI wait → tag
- Comprehensive validation command with 4 areas: QA, Documentation, Release, Security
- Team status report format (`--format team`) with template-based box output
- Report package (`pkg/report`) with structured data types and text/template renderer
- Actions framework with `Run()`, `Propose()`, `Apply()` pattern
- ChangelogAction with schangelog integration
- RoadmapAction with sroadmap integration
- ReadmeAction for badge and version updates
- Interactive mode with `--interactive` flag for Claude Code
- JSON output mode with `--json` flag
- TOON output format (Token-Oriented Object Notation) ~8x more token-efficient
- Claude Code plugin with commands, skills, agents, and hooks
- Gemini plugin support via platform adapters
- SecurityChecker: LICENSE, govulncheck, dependency audit, secret detection
- DocChecker: README, PRD, TRD, release notes, changelog validation
- ReleaseChecker: version format, git status, CI configuration
- Error handling compliance check (detects `_ = err` patterns)
- Git wrapper package with tag, commit, push, status operations
- CI status checking via gh CLI with WaitForCI support
- GoReleaser configuration for multi-platform builds
- Homebrew formula

### Changed

- Module path from `github.com/grokify/prepush` to `github.com/grokify/release-agent`
- Package structure from flat to `cmd/` and `pkg/` layout
- Configuration file from `.prepush.yaml` to `.releaseagent.yaml`

### Documentation

- Updated README with all new commands and features
- Updated PRD with completed feature status
- Updated TRD with new architecture and interfaces
- Added release notes for v0.3.0

## [0.2.0] - 2026-01-04

### Highlights

- Catch more issues locally with new `go build` and `go mod tidy` checks, plus soft warnings that inform without blocking

### Added

- Go check: `go mod tidy` to verify go.mod/go.sum are up to date
- Go check: `go build` to ensure project compiles
- Go check: untracked references detection (soft warning)
- Soft warnings system: checks that report issues but don't fail the build
- Git hook setup instructions with three options (script, symlink, shared hooks)

### Fixed

- Directory walking now correctly handles current directory (`.`)

### Documentation

- Expanded README with git hook setup options and bypass instructions
- Added Go checks detail table documenting hard vs soft checks
- Updated example output to show all 7 checks including warnings

## [0.1.0] - 2026-01-04

### Highlights

- One command to run all pre-push checks across Go, TypeScript, and JavaScript - with auto-detection and monorepo support

### Added

- Multi-language pre-push hook with auto-detection
- Go checks: `gofmt`, `golangci-lint`, `go test`, local replace directive detection
- TypeScript/JavaScript checks: `eslint`, `prettier`, `tsc --noEmit`, `npm test`
- Language detection for Go, TypeScript, JavaScript, Python, Rust, Swift
- Monorepo support with multiple language detection per repository
- Configuration via `.prepush.yaml` with per-language settings
- CLI flags: `--verbose`, `--no-test`, `--no-lint`, `--no-format`, `--coverage`

### Documentation

- README with installation, usage, and configuration examples

### Tests

- Unit tests for detect, config, and checks packages
